Conexiones del hub
Las llaves se guardan solo en este navegador (localStorage). Nunca salen de tu equipo salvo hacia el servicio correspondiente.
Claude API—
Necesaria para generar contenido, iterar copy y diagnosticar ciclos. Se obtiene en console.anthropic.com → API Keys.
GitHub — estrategia .md—
Personal Access Token con scope repo. Sincroniza los documentos de estrategia de cada cliente (lectura y escritura).
Google Sheets — datos de workers—
Endpoint fijo de Ontop, no requiere llave. Trae el snapshot mensual de buckets por cliente.
Cómo funciona el hub
Home → workspace del cliente → flujo de track. Tres niveles, siempre visibles en el breadcrumb debajo de la barra superior.
En el workspace del cliente (sin track seleccionado)
Resumen — métricas del cliente y sus tracks. Elige un track para trabajarlo.
Ajustes — canales (email/WhatsApp), framework de copy, principios de psicología y tono. Se aplican a todos los tracks del cliente.
Estrategia — el documento .md del cliente (desde GitHub) y un chat con todo el contexto para iterar la hipótesis del próximo ciclo.
Al seleccionar un track — flujo de 4 pasos
1 · Generar — Claude crea las cadenas en inglés y español. Requiere la API key configurada.
2 · Editar — revisa cada mensaje, edítalo a mano o pídele cambios a Claude. Exporta el documento Word del cliente.
3 · Envíos — marca qué mensajes ya se enviaron.
4 · Resultados — registra el ciclo (migraciones, apertura, clicks) y pide un diagnóstico a Claude.
Las migraciones se calculan solas comparando el snapshot de Sheets del mes actual contra el anterior. Los aprendizajes de cada ciclo se guardan en el .md de GitHub para que nada se pierda.
